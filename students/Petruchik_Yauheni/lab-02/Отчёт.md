## Цель работы

Спроектировать архитектуру основного сервиса системы планировщика семейного бюджета с использованием гексагональной (hexagonal) архитектуры: структура проекта, порты (интерфейсы), изоляция слоёв.

---

## Вариант №49 — «Про рубли» 💵

**Питч:** Все траты видны, ссоры в семье меньше.

**Ядро домена (фрагмент):** члены семьи, транзакция, лимит категории, кошелек.

**Выбранный сервис:** **Expense Service** — регистрация расхода и проверка на превышение установленного семейного лимита.

---

## Ход выполнения работы

### Часть 1. Архитектурная диаграмма

**Описание сервиса:** Expense Service принимает данные о новой транзакции, сопоставляет её с лимитом категории через доменную модель и сохраняет состояние через адаптер репозитория.

**Направление зависимостей:** `infrastructure` → `application` → `domain` (бизнес-логика лимитов не зависит от типа БД или API).

---

### Часть 2. Структура проекта (скелет)

**Технология:** Python 3.

```
lab-02/
├── Отчет.md
└── src/expense_service/
├── domain/models/
│   ├── budget_limit.py      # Логика проверки превышения
│   └── transaction.py       # Сущность расхода
├── application/
│   ├── port/in_bound/create_expense_use_case.py
│   ├── port/out_bound/budget_repository.py
│   └── service/expense_manager_service.py
└── infrastructure/
├── adapter/in_bound/budget_api_controller.py
├── adapter/out_bound/postgres_budget_repository.py
└── config/container.py  # Dependency Injection
```

---

### Часть 3. Domain Layer

**Ключевая сущность:** `BudgetLimit` — метод `is_exceeded` инкапсулирует математику проверки остатка средств без обращения к внешним API.

---

### Часть 4. Application Layer (порты)

**Входящий порт:** `CreateExpenseUseCase` — контракт сценария добавления новой траты.

**Исходящий порт:** `BudgetRepository` — интерфейс для получения текущих лимитов (`get_limit_by_category`) и записи транзакций.

**Сервис:** `ExpenseManagerService` — координация: получение лимита, валидация через домен, сохранение в базу.

---

### Часть 5. Infrastructure Layer

**Входящий адаптер:** `budget_api_controller.py` — REST API (FastAPI) для мобильных устройств членов семьи.

**Исходящий адаптер:** `PostgresBudgetRepository` — реализация хранения данных в PostgreSQL через SQLAlchemy.

---

### Часть 6. Dependency Injection

**Файл:** `src/expense_service/infrastructure/config/container.py`

Контейнер связывает `ExpenseManagerService` с конкретным экземпляром репозитория, обеспечивая инверсию зависимостей.

---

## Описание портов и адаптеров

| Тип | Название | Назначение |
|-----|----------|------------|
| Входящий порт | `CreateExpenseUseCase` | Сценарий регистрации расхода |
| Исходящий порт | `BudgetRepository` | Абстракция хранилища бюджета |
| Входящий адаптер | Budget API | HTTP контроллер для приема трат |
| Исходящий адаптер | `PostgresRepository` | Реальная работа с БД |
