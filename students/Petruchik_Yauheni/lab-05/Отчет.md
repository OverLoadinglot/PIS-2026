## Цель работы

Реализовать инфраструктурный слой для семейного бюджетного планировщика: адаптеры репозитория, REST API и соединение с БД.

**Питч:** Все траты видны, ссоры в семье меньше.

**Ядро домена:** Члены семьи, Категории, Операции, Лимиты, Отчёты, Совместные расходы.

---

## Ход выполнения работы

### 1. Infrastructure Adapter — Repository

Реализован адаптер `BudgetRepositoryImpl` на SQLAlchemy, работающий с PostgreSQL. Он хранит состояние бюджета в таблице `budgets` и поддерживает методы `get_by_id` и `save`.

### 2. REST API

Создан контроллер на FastAPI:

- `POST /budgets/` — создание бюджета;
- `GET /budgets/{budget_id}` — получение бюджета;
- `GET /health` — проверка доступности сервиса.

API описано через OpenAPI.

### 3. База данных и Docker Compose

Поднимается PostgreSQL через `docker-compose.yml`. Миграции для таблицы `budgets` находятся в `infrastructure/migrations/versions/`.

### 4. Интеграционный тест

Написан тест, который проверяет сохранение и чтение бюджета через репозиторий.
