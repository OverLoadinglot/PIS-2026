## Цель работы

Реализовать CQRS в бюджетном планировщике: разделить write и read модели, синхронизировать read model через события.

---

## Вариант №49 — «Планировщик бюджета семьи “Про рубли”» 💵

**Питч:** Все траты видны, ссоры в семье меньше.

**Ядро домена:** Члены семьи, Категории, Операции, Лимиты, Отчёты, Совместные расходы.

---

## Ход выполнения работы

### 1. Write Model

Write Model работает с агрегатом `FamilyBudget`, сохраняет транзакции и поддерживает бизнес-инварианты.

### 2. Read Model

Реализована денормализованная модель `BudgetSummaryView` для быстрых запросов:

- `budget_id`
- `name`
- `members_count`
- `categories_count`
- `transactions_count`

### 3. Event-Driven Sync

Проекция `BudgetProjection` обновляет read-модель при событии `TransactionAddedEvent`.

### 4. Оптимизация запросов

SQL-скрипт создает индекс на `budget_id`, чтобы ускорить выборку агрегированных данных.

### 5. Тесты проекции

Проверка: при событии `TransactionAddedEvent` обновляется модель `BudgetSummaryView`.
