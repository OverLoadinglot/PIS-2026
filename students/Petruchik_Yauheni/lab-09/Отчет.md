## Цель работы

Реализовать межсервисное взаимодействие через gRPC: контракт в Protocol Buffers, сервер, клиент и потоковую передачу данных.

## Ход выполнения работы

### 1. Протофайлы

Определены сообщения и сервисы в файле `proto/request_service.proto`.

### 2. Генерация кода

Используются `grpcio-tools` для генерации модулей Python: `generated/request_service_pb2.py` и `generated/request_service_pb2_grpc.py`.

### 3. gRPC Server

Сервер реализован в `grpc/server.py` и хранит состояние в памяти. Поддерживаются unary-методы и streaming API для списка активных расходов.

### 4. gRPC Client

Клиент в `grpc/client.py` вызывает методы сервера и может получать потоковые обновления.

### 5. Server-side streaming

Метод `StreamActiveBudgetEntries` отправляет текущие активные бюджетные записи в поток.

### 6. Тесты

Файл `tests/test_grpc.py` проверяет контракт и потоковую передачу.
